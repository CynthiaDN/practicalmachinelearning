classification model: random forest

lets try a random forest-model. This model can handle multicollinearity.

short discription: the random forest-model creates a lot of trees. Each tree is build on a different sample of the data (bootstrapping). A each node of the tree is using a random set of m variables that may contribute to the split. The trees are voted to predict an outcome.

Because its very slow we use clusters.

The chosen resampling method is cross validation (10-fold).

cv 5-fold: accuracy 0,99 almost the same as 10-fold. for k=10 the bias-variance balance is a little bit beter. also cv 30-fold has almost the same accuracy The model seems to be stable. I use k=10.

set.seed(123)
cluster <- makeCluster(detectCores() - 1) # convention to leave 1 core for OS
registerDoParallel(cluster)


fitControl <- trainControl(method = "cv",
                           number = 10,
                           allowParallel = TRUE)

modFitRf <- train(classe ~ ., method = "rf", data=train, trControl=fitControl)

stopCluster(cluster)
registerDoSEQ()
